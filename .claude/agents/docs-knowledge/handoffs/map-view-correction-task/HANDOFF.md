## 構造正本（architecture）— `#device` / `.device-screen` 内の唯一の正

**比較対象はこのツリーのみ。** `.annotations-col`・`.anno-bar`・`#btn-ja` / `#btn-en` / `#btn-b` / `#btn-a` は **本番に移植禁止**（検証 UI）。

### ブロック順序（上から下・変更不可）

1. **`.map-strip`**（高さ 72px 相当）  
   - 子: **`.close-fab`** のみ（モック上は地図画像なしのプレースホルダ可。本番は Leaflet を **この帯の高さにクリップ**する等で「薄帯」に収めるのが MVC-01）。
2. **`.sheet`**
   1. **`.l1-head`** — `.l1-head__main`（`.l1-title`, `.l1-meta`）＋ **`.btn-next-sm`**
   2. **`.b-glance`**（`#b-glance`）— `.b-glance__icon` + `.b-glance__main` > `.b-glance__line1`（数値は内側 **`.l2-v`**) + **`.b-glance__badge`**
   3. **`.day-chips-wrap`** > **`.day-chips`**（`role="tablist"`）— 子は複数 **`button.day-chip[role=tab]`**。各チップは **第1子 `span`（相対ラベル）＋第2子 `span.day-chip__cal`（暦日 M/D）＋第3子 `strong`（降雪 `N cm` / `—`）** の **3 段固定**（**MVC-06**）。
   4. **`.l2`**（`#l2-block`）— `.l2-row` × 3（A 相当の密度ブロック。B 表示時は `hidden` でよいが **ノードは残す**）
   5. **`.tl-legend`**
   6. **`.tl-scroll`** > **`.tl-slot`** × N（`role="list"` / `role="listitem"`。帯クラス `band-prev` / `band-day` / `band-next`、`is-wet` 等）

### B / A（本番での実装契約）

- モックの B/A ボタンは **置かない**。**`.b-glance` と `.l2` の `hidden` 反転**だけで注釈トグルと同じ見え方にする（`setMode` と同型）。
- **「ひと目」「詳細」等のラベル UI をデバイス内に新設しない**（MVC-03）。

### タイムライン（L3）の位置づけ

- **誤解禁止**: L3（`.tl-legend` + `.tl-scroll`）は「日付チップの直下」ではなく、**正本 DOM では `.l2` の直下**（＝チップと L3 のあいだに **常に `.l2` ブロックが存在**）。B モードで `.l2` が非表示でも **DOM 順序は維持**する。

### スロット・i18n（実装契約）

- **`.tl-slot__label`**: 画面上は略号可。**`aria-label`（または `data-slot-aria` 経由）に完全形**（mock の `sa0`… と同型）。EN は **`html lang="en"`（または `.device-screen[lang="en"]`）で `--slot-w`・2 行クランプ・バッジ折返し**（mock-016 は 013 制度を継承）。
