## 4. 次にやること（提案・詳細）— **No.1b（A-1 コード）**

**位置づけ**: 上表で **唯一未完了の「次の一手」**は **1b**（指示書は 1a で完了済み）。

### 4.1 ゴール（ビジュアルと対応）

| 図の意図 | 実装で満たすこと |
|----------|------------------|
| **図A** 左（5チップが一画面） | モバイル幅で **前日〜+3** が **5チップ分の幅**に収まり、**日付が切れない**。 |
| **図A** 右（+4〜+6） | **横スクロール**で **+4 / +5 / +6** に到達。**スナップ**で半チップが残らない。 |
| **図A** 下（開いた直後→確定後） | 初回ペイント後に **チップ幅がビューポートにフィット**（`cqw` または `ResizeObserver` + `--chip-w`）。 |
| **図B** | 表示は **マスク内**のみ。次のチップが **画面端に半分だけ出たまま**にならない。 |
| 共通 | **`targetDayOffset` が 4〜6** のとき、再描画後も **アクティブチップが見える**（`ensureActiveDateVisible` 等）。 |

### 4.2 作業リスト（ファイルは JP / EN 両方）

1. **HTML**  
   - `#top-date-btns` と `#date-btns` を **`date-btns-viewport` でラップ**し、内側を **`date-btns-track`**（既存 id は指示書どおり）。  
   - 参照: `IMPLEMENTATION_ORDER_DATE_SELECTOR_5_SLIDE_OPTION_A.md` §3

2. **CSS**  
   - **`max-width: 700px`**: ビューポート `overflow-x: auto`、トラック `display:flex`、チップ幅 **(ビューポート − gaps) / 5**、`scroll-snap-type` / `scroll-snap-align`。  
   - **`min-width: 701px`**: 従来の **8列 grid**（横スクロールなし）。  
   - 参照: 同指示書 §4

3. **JS**  
   - `renderDateButtonsInto`: 日本語で **`明後日`**（offset 2）、**+3日** など **案Aのラベル**に合わせる（EN も同等の情報）。  
   - 描画後 **`ensureActiveDateVisible`**（+4〜+6 選択時に `scrollLeft` 調整）。  
   - `setTargetDayOffset(0..6)` は **変更しない**；前日 `-1` は **disabled のままで可**。  
   - 参照: 同指示書 §6

4. **受け入れ**  
   - 同指示書 **§8 受け入れ条件**をすべて満たしたら、**§2 の表で 1b を `[x]`** にする。
