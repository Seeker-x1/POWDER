# デプロイ・404・Search Console（リダイレクト）の切り分け

## 結論（先に）

- **`data/*.json` をリポジトリに置き、本番のドキュメントルートへ同じ階層でデプロイする**と、ブラウザの **Console に出る `fetch` の 404 は解消**できます。
- **Search Console の「リダイレクトのためインデックスされない」**は、**HTTP/HTTPS・www の有無・正規 URL のどれを代表にするか**などの **サーバー／DNS 設定**の話であり、**JSON を置いただけでは自動では解消しません**。

両方を別タスクとして扱ってください。

---

## 1. 404 をなくす（このリポジトリと本番の関係）

`ski-powder-hunter.html` / `ski-powder-hunter-en.html` は読み込み時に、**HTML と同じ階層からの相対パス**で次を取得します（例: `https://例.com/ski-powder-hunter.html` なら `https://例.com/data/...`）。

本リポジトリでは、少なくとも次を **コミット済みのファイル名**で用意しています（中身が空オブジェクト `{}` のものは「任意キャッシュが無い」状態で 200 を返すためのプレースホルダです。運用で上書きしてください）。

| ファイル | 用途の概要 |
|-----------|------------|
| `data/weather.json` | 天気キャッシュ（任意） |
| `data/jma-snow.json` | 気象庁積雪（既存） |
| `data/resort-snow.json` | ゲレンデ公表積雪（任意） |
| `data/lift-times.json` | リフト開始（任意） |
| `data/resort-operating.json` | 営業期間上書き（任意） |
| `data/jma-prevday-hourly.json` | 前日時間帯マージ用（任意・形式は README 参照） |

**本番サーバー**では、公開ディレクトリに **`ski-powder-hunter.html` と兄弟の `data/` フォルダごと**アップロードされているか確認してください。GitHub にあっても、FTP 等で **HTML だけ**上げていると 404 のままです。

確認用 URL（本番ドメインに置き換え）:

- `https://www.例.com/data/weather.json` → **200** かつ JSON  
- 同様に他ファイルも直叩きで確認

---

## 2. GSC で「リダイレクト」と出るとき（認識の揃え方）

Google は **最終的に 200 を返す URL** を主にインデックス対象にします。

よくあるパターン:

- `http://www.example.com/` が **`https://www.example.com/` へ 301**  
  → インデックスの主役は **HTTPS 側**。GSC のプロパティも **HTTPS の URL プレフィックス**（またはドメインプロパティ）に合わせる。
- `http` と `https`、`www` と非 `www` が混在  
  → **どれか一つに 301 で統一**し、サイトマップ・内部リンクもその形に揃える。

**JSON の 404 を直しても、上記の「正規 URL のリダイレクト」は変わりません。**  
GSC のメッセージを消したい場合は、**URL 検査で「最終的に到達する URL」**を開き、その URL でプロパティを持つ・サイトマップを出す・`canonical` をその URL に揃える（HTML 側の変更は別タスク）ことが必要です。

---

## 3. DevTools の「リダイレクトっぽいアイコン」について

Network タブで **404 のリクエストの横に矢印アイコン**が付くことがありますが、**Search Console の「ページにリダイレクトがある」と同一とは限りません**。  
まず **ステータスコード（200 / 301 / 404）** と **Response Headers の `Location:`** を見て判断してください。

---

## 4. 運用メモ

- 実データを入れるときは `data/README.md` の各節と `*.sample.json` を参照し、**本番用の値に差し替え**てください。
- `resort-snow.json` 等が `{}` の間は、既存の `RESORTS` や API 取得にフォールバックします（アプリは動きます）。
