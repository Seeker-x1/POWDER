# 実装指示書：モバイル・マップ画面を新レイアウトへ統一

**目的:** 現在のモバイル・マップUIを、提案モックに最も近い形へ寄せる。重なり・閉じられない・情報過多を解消し、地図主役を維持する。  
**参照モック:** `ui-proposal-mobile-map-realistic-required-elements.png`（Cursor生成物）

---

## 1. 最終UIの要件（モバイル幅）

対象: `max-width: 700px`（既存ブレークポイントを踏襲）

1. **上部操作は2段だけ**
   - 1段目: `戻る` + `検索`（同一行）
   - 2段目: `OFF / 3h / 6h`（1行セグメント）
   - 地図キャンバス上に、追加の凡例・説明箱を置かない

2. **地図領域を最大化**
   - 上記2段以外は地図優先
   - 右下は `+/-` のみ（Leaflet標準）

3. **ピン表示は増えても破綻しない**
   - 単独ピン + クラスタ（数値バッジ）
   - 選択中ピンだけ強調（発光/拡大）

4. **詳細は中央ポップアップでなく下シート導線**
   - 画面下にハンドル（または薄いプレビュー）
   - 詳細展開時も、閉じるとすぐ地図パンへ戻れる

---

## 2. 現実実装のルール（見た目だけでなく操作を守る）

### 2.1 重なり禁止

- モバイル中は、次を**同時に重ねない**:
  - 検索/戻るバー
  - JMAトグル帯
  - 詳細ポップアップ（既存）
- 詳細表示中は、上部フロートを退避（`visibility:hidden` + `pointer-events:none` など）

### 2.2 タップしやすさ

- 戻る・検索・トグル各要素は目安 40〜44px 以上のタップ領域
- ピンは最小可視サイズを確保し、密集域はクラスタ化して誤タップを減らす

### 2.3 情報量

- 「説明のための常時パネル」は置かない（`MAP_UI_OVERLAY_POLICY.md` 準拠）
- 必要説明はヘルプ/シート内へ

---

## 3. ピン表示仕様（モック寄せ）

1. **通常ピン**
   - 小型ドロップ/円形いずれでも可
   - 色はスコア帯（例: 高=明色、中=中間、低=暗色）

2. **クラスタ**
   - カウント数字を中央表示（例: 3,7,12）
   - 低ズームで集約、高ズームで分解

3. **選択ピン**
   - 1件のみ視覚強調（グロー/リング/サイズ+）
   - 他ピンは通常状態

4. **パフォーマンス前提**
   - 全件常時DOM重描画を避ける
   - クラスタ利用またはズーム連動表示で負荷を抑える

## 6. 実装メモ（レビュー向け）

- モックの完全一致は不要。優先順は:
  1) 重なり解消
  2) 地図主役
  3) ピン密集耐性
  4) 見た目寄せ
- 既存機能（検索・JMA切替・詳細導線）は落とさない
