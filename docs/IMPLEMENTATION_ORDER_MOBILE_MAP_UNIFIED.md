## 3. 昨日/12h/6h 表示仕様（必須）

## 3.0 上段（日付セレクタ）と下段（当日時間帯）の役割分離（必須）

- **上段の日付セレクタに `前日` を残す。**  
  **レイアウト（案A・モバイル）**: **`IMPLEMENTATION_ORDER_DATE_SELECTOR_5_SLIDE_OPTION_A.md` を正本とする**（**5 チップ幅のビューポート**＋ **+4/+5/+6 は横スクロール**）。本節以前の「8列・横スクロール禁止」は **上段については案Aにより改訂**される。  
  **デスクトップ（幅 >700px）**では、従来どおり **8 列グリッド**でよい。  
  いずれも「追加しただけ」で終わらせず、**はみ出し・極小フォントによる可読性喪失**を禁止する。
- 下段の当日時間帯マス列も、前日2枠を含めて8マス表示する。
- つまり本件は **上段8列 + 下段8マス** を両立させる。

## 3.1 表示位置

- **全日タブ共通（上段日付セレクタ）**:
  - 列順（論理）は `前日 | 今日 | 明日 | 明後日 | +3 | +4 | +5 | +6` とする。
  - **モバイル（≤700px）の表示**は **`IMPLEMENTATION_ORDER_DATE_SELECTOR_5_SLIDE_OPTION_A.md`**（**5 可視＋スライド**）。
  - **デスクトップ**では **8 列を一画面**に収める（横スクロールなし）か、案Aと同じトラックを広い画面で展開してよい。
  - 画面幅に収めるための **極小フォントのみ**に頼らない（案Aの目的）。
  - 推奨の最小値目安:
    - タップ領域高さ: 40px 以上
    - ラベル文字: 11px 以上
    - 値文字: 16px 以上
    - 行間/余白: 4px グリッド基準

- **全日タブ共通（下段マス列）**:
  - `前日合計` は当日情報の左側（横並び）に表示。
  - 時間帯バー左端に次の 2 本を追加し、**合計8マス**にする:
    1. `前日 12–18時`
    2. `前日 18–24時`
  - その後に既存の `0–5時` 以降を続ける。
  - 8マスに収めるため、既存マス幅・余白・フォントを縮小して横並びを維持する（折返し禁止）。

## 3.2 ラベルの意味（固定）

- ラベルは **JST カレンダー前日固定**。
- 閲覧時刻に依存する相対ラベル（「12時間前」「6時間前」）は使わない。

## 3.3 データソース（タブで切替）

- `offset=0`（今日）:
  - 気象庁の観測系データを使用すること。
  - `前日合計`, `前日 12–18時`, `前日 18–24時` は同一ルールで算出。
  - Open-Meteo で代替しない。
- `offset>=1`（明日〜+6）:
  - 表示中タブ `D` に対して、`D-1` の予測値を前日表示として使う。
  - 例: `明日` は `今日予測`、`明後日` は `明日予測` を前日として表示。

## 3.4 降雪換算ルール（必須）

- **降水量 + 気温条件**で降雪相当量に解釈する。
- ルールは画面内で矛盾なく同一適用する。
- 欠測は `—` などで明示し、値を捏造しない。
- `offset>=1` 側（予測）でも、同じ換算ルールを適用する。

## 7. 8列化のUI許容ライン（デザイナー基準）

### 7.1 結論

- **8列化のための縮小は許容**する。  
  ただし「可読性・操作性」を下回る縮小は不許可。

### 7.2 圧縮の順序（必須）

1. **余白を先に圧縮**（カード内padding / カード間gap）
2. 次に **中段ラベル**を1段階縮小
3. 最後に **上段日付**を縮小
4. **値（cm）は最小縮小**に留める（視認性最優先）

### 7.3 最小サイズ基準（これ未満は破綻）

- タップ領域高さ: **40px以上**
- 中段ラベル: **11px以上**
- 上段日付: **10px以上**
- 値（cm）: **16px以上**
- カード間gap: **4px以上**

---

## 8. 英語版（文字数増加）対応

英語版は文字数超過で破綻しやすいため、**ラベル短縮を先に適用**する。

### 8.1 推奨短縮（英語）

- `Yesterday` → `Prev`
- `Today` → `Today`
- `Tomorrow` → `Tmrw`
- `+2 days` → `+2d`
- `+3 days` → `+3d`
- `+4 days` → `+4d`
- `+5 days` → `+5d`
- `+6 days` → `+6d`

曜日は短縮可（ユーザー合意済み）。

- `Sun` / `Mon` / `Tue` / `Wed` / `Thu` / `Fri` / `Sat`

### 8.2 省略優先順位（英語）

1. 曜日を3文字短縮（`Sun` 等）
2. 日数ラベルを `+Nd` へ統一
3. それでも厳しい場合のみフォントを1段縮小

### 8.3 禁止

- 英語だけ横スクロールで逃がす
- 英語だけ行折返ししてカード高さを崩す

## 9. 破綻が出た場合の修正案（先に提案してから実装）

以下のどれかを満たしそうなら、実装担当は勝手に確定せず2案提示すること。

- 文字重なり
- カード内クリップ
- タップ領域40px未満
- 横スクロール発生（※上段の**案A以外**の破綻で発生する場合）

### 修正案テンプレ

- **案A（推奨）:** 余白再配分 + ラベル短縮（英語は `Prev/Tmrw/+Nd`）
- **案B（次善）:** カード角丸や枠線を軽量化して内寸を増やす

両案でスクショ比較（JP/EN 各1枚）を提出し、採用案を決める。
