# 実装指示書：モバイル狭幅でのトップ画面はみ出し・見切れの防止

**ステータス**: 実装用  
**対象画面**: `#view-top`（パウダーランキング・週間8マス・ツールバー・TOP10一覧）  
**関連**: `IMPLEMENTATION_ORDER_MOBILE_TOP_VISUAL_IMPLEMENTATION.md`（全体ゴール）、本書は **狭幅レスポンシブ**に特化。

---

## 1. 問題

細い端末（小型スマホ・縦向き）で次が起きる:

- 画面右端が **見切れる**／**横スクロールがページ全体に出る**
- 週間8マス・ツールバー・カード内メタ行が **幅を取りすぎ**てレイアウトが破綻する
- （別件）縦 `flex` により **ヒーロー領域が潰れ**、帯と一覧が **重なって見える**ことがある

本指示書は **CSS 中心**で、**一般的なスマホ幅で「横方向の破綻を起こさない」**ことをゴールとする。

---

## 2. スコープと禁止事項

### 2.1 やること

- **幅 320px 以上 ～ 430px 前後**（一般的なスマホ）で、トップ画面が **横にはみ出さない**ようにする。
- 意図的な **横スクロールは「週間8マス帯」内に限定**できるようにする（§4）。
- **ツールバー**・**ランキングカード**が狭幅で **折り返し・省略**され、読める状態を保つ（§5・§6）。

### 2.2 やらないこと

- **PC（広い画面）のレイアウトを変えない。** 変更は **`@media (max-width: 700px)`**（またはプロジェクト既存のモバイルブレークポイント）**内のみ**。  
  - 既存で `700px` を使っているならそれに合わせる。未使用なら **701px 以上に影響しない**新規メディアクエリを追加する。
- **320px 未満**を「公式サポート」とはしない（§7 限界）。

### 2.3 触るファイル（想定）

- `ski-powder-hunter.html` 内の `<style>`（または該当 CSS ファイル）

---

## 3. 受け入れ条件（必須）

| ID | 条件 |
|----|------|
| A1 | DevTools または実機で **幅 320px・375px・390px** の各幅で、**`#view-top` を含むページにページ全体の横スクロールが出ない**（`document.documentElement.scrollWidth <= clientWidth` で確認可）。 |
| A2 | 週間8マスは **右端が見切れない**か、**見切れる場合は帯ラッパー内だけ**横スクロールできる（§4.2）。 |
| A3 | ツールバー（並び替え・エリア・更新）が **はみ出して不可視にならない**（折返し・省略・縮小のいずれか、§5）。 |
| A4 | ランキングカード内の **メタ行**（降雪・風・気温）が **スコア数字と致命的に重ならない**（§6）。 |
| A5 | **701px 以上**のウィンドウでトップを開き、**本変更前と目視で同程度**である（意図しない PC レイアウト変更がない）。 |

---

## 4. 週間8マス（`tw-strip-grid` / `.top-target-week-strip`）

### 4.1 方針（どちらかを採用し、指示書にコメントで明記）

**推奨: 案 A — 帯内横スクロール**

- **`.top-target-week-strip`**（または週間グリッドの直接の親）に  
  `overflow-x: auto`  
  `-webkit-overflow-scrolling: touch`  
  `scroll-snap-type: x mandatory`（任意）  
  左右の **ネガティブマージンでフルブリード**する場合は、**ページ幅を超えない**範囲に留める。
- **`.tw-strip-grid`** は `min-width` を **8列分確保**（例: `min-width: min(100%, 8 * 44px + gap)` など）し、**親の `overflow-x: auto` の中だけ**が横に動く。

**代替: 案 B — 8列をすべて表示域に収める**

- `gap`・セル `padding`・フォントを **320px で再計算**し、**8列が収まる**まで縮小。  
- **可読性が落ちる**ため、**案 A が優先** unless ディレクターが B を指定。

### 4.2 CSS 要件（モバイルメディアクエリ内のみ）

- `.tw-strip-cell` に **`min-width: 0`**（grid 子のはみ出し防止）。
- グリッドは **`grid-template-columns: repeat(8, minmax(0, 1fr))`** を維持しつつ、**案 A** のときは **グリッド全体の最小幅**で帯がスクロールする。

### 4.3 ヒーロー潰れ対策（重なりが再発する場合のみ・モバイル限定）

以下は **`@media (max-width: 700px)` 内のみ**に書く:

```css
#view-top.view-top .top-hero {
  flex-shrink: 0;
  min-height: min-content;
}
#view-top.view-top .top-hero.top-hero--static {
  min-height: min-content;
}
#view-top.view-top .top-ten-wrap {
  min-height: 0;
  position: relative;
  z-index: 1;
}
```

（**701px 以上では追加しない**。）

## 5. ツールバー（`.top-toolbar`）

### 5.1 要件

- 320〜375px で **3要素が同時に1行**で厳しい場合:
  - **エリア表示**に `flex: 1; min-width: 0; overflow: hidden; text-overflow: ellipsis; white-space: nowrap;`
  - **並び替え**の `select` に `max-width`（既存があれば維持）＋必要なら `min-width: 0`
  - または **`flex-wrap: wrap`** で **2行目に「更新」**だけ回す、など **一貫したルール**を1つ選ぶ。

### 5.2 禁止

- ツールバーだけが **ビューポート外に押し出される**状態。

---

## 6. ランキングカード（`.view-top .top-card`）

### 6.1 要件

- `.top-card` は **`min-width: 0`**（子の flex で横膨張しない）。
- `.top-card-body` に **`min-width: 0`**。
- `.top-card-meta` は **`flex-wrap: wrap`**、**`gap`** 維持、フォントはモバイルのみ **1段階小さくてよい**（例: `12px` → `11px`）。
- 右のスコア列（`.top-card-score`）は **`flex-shrink: 0`** のまま、**左本文が折り返しても重ならない**こと。

### 6.2 確認

- 長いゲレンデ名・長いメタ行でも **スコアが被らない**こと。

---

## 7. 限界値（仕様として固定）

| 項目 | 値 |
|------|-----|
| 最低ターゲット幅 | **320px**（これより狭い端末はベストエフォート） |
| 週間帯のセル最小幅（案A採用時） | **36〜44px** の範囲で調整（タップ領域確保） |
| ブレークポイント | **`max-width: 700px`** でモバイル専用ルールを閉じる（PC非変更） |

---

## 8. テスト手順（実装者が実施）

1. Chrome DevTools レスポンシブ: **320 / 375 / 390** 幅でトップを表示。  
2. 各幅で **横スクロールの有無**（ページ全体）を確認。  
3. 週間帯を **指で横スワイプ**し、**8マスすべてに到達できる**こと（案A）。  
4. **幅 1280px** 前後でトップを表示し、**PCレイアウトが意図せず変わっていない**こと（A5）。

---

## 9. 完了定義

§3 の **A1〜A5** をすべて満たし、レビュアが **モバイル3幅＋PC1幅**で確認して承認すること。
