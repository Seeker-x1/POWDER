## 1. 週間グラフ（降雪予測ベース）とターゲット日

### 1.1 現状

- リゾートカード内に **8 列**（前日実績/合計＋オフセット 0〜6 日）の降雪バーがある（`renderList` 内 `barsHtml`）。
- トップのランキングビュー（`#view-top`）には **同種の週間グラフはない**。日付は `renderDateButtonsInto("top-date-btns")` のチップのみ。

### 1.2 ゴール（ビジュアル準拠・**1本化**）

- **ターゲット日の選択 UI** と **週間の降雪見立てグラフ**は **別ブロックに分けない**。**横一列の単一コンポーネント**（「一本」）にまとめる。
- その **1 本**の中を **8 マス**で構成する: **前日（Prev day）** ＋ **7 日分**（いわゆる今日〜+6 に相当する予報日）。各マスに **(1) 日付の 2 行ラベル**（例: `3/22` / `Sun`）**(2) 降雪見立ての棒** を **同じ列**に置き、**ターゲット日の選択**はこの列のタップで行う（前日は **選択不可**・参照のみ、既存方針どおり）。
- 別途「5 チップだけの日付行」をトップに **重ねて置かない**（マップ側など既存仕様は別タスクで整合を取る）。

### 1.3 データソース（集約ルール）— **確定**

- **案 A 採用**（ディレクター決定）: **表示中ランキング（最大 10 件）**について、日ごとに `getSnowfallAdjusted` / 前日は `getPrevDayDisplayForResort` の値を取り、**中央値**を TOP の 8 マスストリップの棒の高さに使う。
- 詳細は **`IMPLEMENTATION_ORDER_MOBILE_TOP_VISUAL_IMPLEMENTATION.md` §6** を正とする。

**受け入れ**: ツールチップ等に **集約の説明文**があること。一覧の絞り込み・ソート変更でストリップが **連動**すること。

### 1.4 はみ出し・横長の防止 — **8 マスは入り切るか**

**結論**: **正本ビジュアルと同じ「2 行ラベル＋細い棒」**に揃えれば、**多くの端末で入り切らせられる**。ただし **8 列は 7 列より厳しい**ので、**実装は「320px で必ず受け入れ」＋「フォールバック明記」**が必須。

| 観点 | 内容 |
|------|------|
| 幅の目安 | コンテンツ幅 320px 時、**8 等分**は列あたり **約 40px**。ラベルは **必ず 2 行**（日付 / 曜日）。**1 行に `3/22 Sun` を詰めない**（はみ出しの主因になりやすい）。 |
| レイアウト | `grid-template-columns: repeat(8, minmax(0, 1fr))` または `flex`＋子 `min-width: 0`。**`nowrap` をセル全体にかけない**。 |
| タップ | 列幅が狭いので **縦方向のタップ領域**（セル全体の高さ）を確保し、**44px 相当**を狙う（高さで補う）。 |
| フォールバック | それでも破綻する場合は **この 8 マス帯だけ** `overflow-x: auto` ＋ `scroll-snap-type: x mandatory` とし、**セル最小幅**（例: 44px）を確保して **横スクロール**（ページ全体はスクロールさせない）。 |

**過去に「横長・文字が入らない」になった原因の典型**: 8 列なのに **1 行ラベル**・**`min-width` 付き固定幅**・**gap 大きめ**の併用。正本どおり **2 行＋ `minmax(0,1fr)`** で再発を避ける。

## 2. コントロール行（ビジュアル同等）

### 2.1 ゴール

- **並び替え**（ドロップダウン）・**エリア**（現在の選択を示す 1 行）・**更新** を **1 行**に配置（`docs/mobile-ux-images/top-ui-v9-weekly-dates-en-title.png` 系と同等）。
- 狭い幅では **折返し禁止**ではなく、**折返し2行目**または **エリア省略**（`text-overflow: ellipsis`）を許容するが、**タップ領域は 44px 相当を確保**。

### 2.2 現状との差分

- 現状: `top-sort-row` と `btn-top-refresh` と `top-region-pills` が **縦に分離**。
- 対応: DOM を **1 ツールバーコンテナ**にまとめ、**順序**は「Sort | Area | Refresh」。

## 3. エリア変更＋固定＋更新＋再訪問時の保持

### 3.1 ゴール

- エリアは **ピル多数**ではなく **「エリアを変更 › 地域名」1 行**（または同等の単一行 UI）。タップで **ボトムシート / モーダル**（既存方針があればそれに従う）。
- **更新**ボタンは常に同じ位置にあり、**押下で `renderTop10()`（および週間データの再取得）**。
- **ページ再読込後も**、選択エリア・並び替えキーを **復元**する。

### 3.2 永続化

- `localStorage` にキーを定義（例）:
  - `powder.topRegion` — 地域 ID またはコード
  - `powder.topSort` — `powder` | `snow` | `depth` | `wind` | `temp`
  - `powder.lang` — `ja` | `en`（後述の一元化と整合）
- 初回はデフォルト（全国 / 既定ソート）とし、**書き込みはユーザー操作時のみ**（自動上書きでバックエンド仕様を壊さない）。

## 5. i18n 一元化（5・6 共通）

### 5.1 ゴール

## 7. 実装順序（推奨）

1. **i18n 基盤**（文字列オブジェクト＋単一 HTML 化の土台）※大きい変更は先にブランチ。
2. **TOP レイアウト**（動画除去、ダーク、ツールバー 1 行化）。
3. **`localStorage` 読み書き**（エリア・ソート）。
4. **8 マス一本化ストリップ**（前日＋7 日・集約ルール・320px 受け入れ・§1.4 フォールバック）。
5. **`targetDayOffset` と列ハイライトの同期**（前日列は常に disabled 表示）。
6. 回帰テスト（日付 5 スライド、JMA 前日チップ、マップ連携）。
