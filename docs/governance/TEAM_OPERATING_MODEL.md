# Team Operating Model

## Roles

- Product Owner (1): KPI and priority owner.
- Front/Impl (1-2): Mobile-first implementation and measurement.
- Content Ops (1): JP/EN copy quality and consistency.
- SEO/Growth (1): Search strategy and weekly optimization.
- Ops/Community (shared): User feedback loop and distribution.

## RACI (simple)

- **Define source of truth**: PO (A), Front (R), others (C)
- **Implement UI changes**: Front (A/R), PO (C), Content (C)
- **JP/EN copy alignment**: Content (A/R), Front (C), PO (C)
- **Release gate decision**: PO (A), Front (R), Content/SEO/Ops (C)

## Weekly Cadence

1. Monday: PO sets one source-of-truth task.
2. Mid-week: Front submits implementation evidence.
3. Friday: Cross-role review, approve or return with one-line defects.
