# 気象庁「現在の積雪の深さ」をゲレンデに反映する方法

## 概要

[気象庁の積雪の深さ一覧表](https://www.data.jma.go.jp/stats/data/mdrr/snc_rct/alltable/snc0316.html)（CSV: [snc00_rct.csv](https://www.data.jma.go.jp/stats/data/mdrr/snc_rct/alltable/snc00_rct.csv)）は**観測所単位**のデータで、ゲレンデとは直接紐づいていません。  
「どのゲレンデに、どの観測所の積雪を表示するか」を決める必要があります。

## 紐づけ方針：**「最も近い積雪観測所」で紐づける**

- 気象庁の積雪CSVには **観測所番号**（アメダス観測所番号）・都道府県・**地点名**・**現在の積雪の深さ(cm)** が含まれますが、**緯度・経度は含まれません**。
- 一方で、**アメダス観測所一覧**は別途、**緯度・経度付き**で公開されています。
  - 例: [LANDWATCH アメダス観測所一覧](https://landwatch.info/topic/ame-code/)、[opendataapi.jp アメダス観測所一覧](https://opendataapi.jp/data/providers/jma/datasets/amedas_master/latest/)、[washitake.com アメダス観測所一覧](https://washitake.com/weather/amedas/obs_stations.md) など。
- そこで次のようにします。
  1. **観測所番号 ↔ 緯度・経度**の対応表（`data/amedas-stations.json`）を用意する。
  2. 各**ゲレンデ**は既に `lat` / `lng` を持っているので、**「そのゲレンデから最も近い、積雪CSVに登場する観測所」**を1つ選ぶ（距離は Haversine などで計算）。
  3. その観測所の「現在の積雪の深さ」を、そのゲレンデの**積雪（気象庁）**として表示する。

これで「エリア」は**距離で自動的に**決まり、都道府県や地域名で手で振る必要はありません。

## 気象庁データの更新時刻（取得タイミングに合わせられる）

- **積雪の深さ・3時間降雪量（雪の状況）** は **毎時50分頃** に更新されます（[データの更新時刻](https://www.data.jma.go.jp/stats/mdrr/man/update_n.html)）。
- つまり 0:50, 1:50, 2:50, … 23:50（JST）頃に、直前の「毎時00分」観測データが反映されます。
- 取得を更新スケジュールに合わせる場合の例:
  - **毎時1回拾う**: 毎時 55 分 JST に実行（例: cron `55 * * * *`、タイムゾーン Asia/Tokyo）
  - **日数回で十分な場合**: 例として 8:55, 12:55, 18:55 JST など
  - **1日1回でよい場合**: 例として 8:55 または 9:00 JST

## データの流れ（1日1回 or 毎時）

1. **スクリプト**（例: `scripts/fetch-jma-snow.js`）を、上記の更新時刻の直後に実行する（例: GitHub Actions で毎時55分 JST または日1回 9:00 JST）。
2. スクリプトは以下を行う:
   - 気象庁の積雪CSV（`snc00_rct.csv`）を取得する。
   - `data/amedas-stations.json` を読み、観測所番号 → 緯度・経度の対応を得る。
   - 各ゲレンデについて、CSVに存在しかつ `amedas-stations.json` に座標がある観測所のうち、**最も近い観測所**を選ぶ。
   - その観測所の「現在の積雪の深さ」を、そのゲレンデ用に保存する。
3. 出力例: `data/jma-snow.json`  
   `{ "resort_id": { "depth_cm": 78, "station_name": "倶知安", "station_no": 16217, "observed_at": "2025-03-16T07:00:00+09:00" } }`
4. フロントでは `data/jma-snow.json` を読み、存在するゲレンデには**気象庁の積雪**を表示し、無い場合は従来どおり Open-Meteo の積雪（モデル推定）を表示する。

## 観測所一覧（緯度・経度）の用意

- 積雪CSVの「観測所番号」は**アメダス観測所番号**と一致します。
- **アメダス観測所一覧**（観測所番号・地点名・緯度・経度）をどこかから取得し、`data/amedas-stations.json` の形式に変換して配置します。
- 形式例:  
  `{ "16217": { "lat": 42.9, "lng": 140.75, "name": "倶知安" }, ... }`
- リポジトリには**サンプル用の少数地点**だけ入れておき、本番では以下いずれかで**全観測所**を用意することを推奨します:
  - LANDWATCH 等のCSVをダウンロードし、観測所番号・緯度・経度・地点名を抽出してJSONに変換する。
  - [気象庁 地域気象観測所一覧表](https://www.jmbsc.or.jp/jp/offline/fd0080.html) などから同様に変換する。

## 注意点

- 積雪観測を行っているのは**アメダスだけではない**ため、CSVにだけ登場する観測所（座標一覧に無い）は、距離計算の対象にできません。その場合は「そのゲレンデには気象庁積雪を表示しない」か、必要に応じて手動で座標を `amedas-stations.json` に追加します。
- 気象庁のCSVは**日付でURLが変わる**場合があります。公式ドキュメントの [CSVダウンロードについて](https://www.data.jma.go.jp/stats/data/mdrr/docs/csv_dl_format_snc.html) で「最新のCSVファイル」のURLを確認してください（例: `snc00_rct.csv` が固定のこともあります）。

## 降雪量（3時間・観測値）の追加

- **積雪の深さ**（snc）は「今の積雪何cm」、**降雪量**（snd）は「直近3時間で何cm降ったか」の**観測値**です。予報ではなく実際の数値です。
- 気象庁の [3時間降雪量一覧表](https://www.data.jma.go.jp/stats/data/mdrr/snc_rct/alltable/snd3h0316.html) および [降雪量全要素CSV](https://www.data.jma.go.jp/stats/data/mdrr/snc_rct/alltable/sndall00_rct.csv)（sndall00_rct.csv）を同じ「最寄り観測所」で紐づけて取得しています。
- `scripts/fetch-jma-snow.js` は積雪CSV（snc00_rct.csv）に加え、降雪量CSV（sndall00_rct.csv）を取得し、`jma-snow.json` に `snowfall_3h_cm`・`snowfall_3h_observed_at` を追加します。冬期以外は降雪量CSVが取れない場合があります。
- フロントでは「3h観測 XXcm」「直近3h観測降雪 XXcm」として表示します（いずれも気象庁観測＝予想ではない数字）。

## 日最深積雪（日最大深）の追加

- 気象庁の [日最深積雪ランキング](https://www.data.jma.go.jp/stats/data/mdrr/rank_daily/data0316.html#snc)（「日最深積雪（5cm以上のみ）7時00分の値」）と同じ指標を反映するため、[最深積雪CSV](https://www.data.jma.go.jp/stats/data/mdrr/snc_rct/alltable/mxsnc00_rct.csv)（mxsnc00_rct.csv）を取得し、同じ「最寄り観測所」で紐づけています。
- `jma-snow.json` に `depth_max_today_cm`（当日の最深値cm）・`depth_max_at_hhmm`（最深値の時刻）・`depth_max_observed_at` を追加します。
- フロントでは「日最深 XXcm」バッジ・ポップアップ行で表示し、**積雪量ランキング**のソートでは日最深を優先して使用するため、気象庁の日最深積雪ランキングと並びが一致します。

## 気象庁に紐づかないゲレンデの積雪（ゲレンデ公表の集計）

多くのゲレンデは公式サイトで「本日の積雪 ○○cm」を公表しています。気象庁の観測地点に紐づけないゲレンデでも、この**ゲレンデ公表値**を集計して表示・ランキングに使えます。

### 優先順位（表示・積雪量ランキングとも）

1. **気象庁**（jma-snow.json に紐づいた観測所の値）
2. **ゲレンデ公表** … 次のいずれか  
   - `data/resort-snow.json` にそのゲレンデの ID で記載した値  
   - 上記が無い場合は RESORTS の **snowDepth**（HTML 内の静的値・要更新）
3. **Open-Meteo モデル**（予報モデルによる推定）

### resort-snow.json で集計する方法

- **形式**: `{ "リゾートID（数値の文字列）": { "depth_cm": 数値, "updated_at": "YYYY-MM-DD" }, ... }`
- **例**: `{ "45": { "depth_cm": 315, "updated_at": "2025-03-16" }, "12": { "depth_cm": 180, "updated_at": "2025-03-16" } }`
- **更新のしかた**  
  - **手動**: 各ゲレンデの公式サイトを確認し、ID と積雪を追記・更新する。  
  - **スクリプト**: ゲレンデ公式ページをスクレイプして JSON を生成するスクリプトを用意し、cron や手動で実行する。サイトごとに HTML が違うため、代表的な集約サイト（例: スキー場.com、 gelanding 等）から取得するか、主要ゲレンデだけ対応する形が現実的です。
- ファイルが無い、または該当 ID が無い場合はスキップされ、次に RESORTS.snowDepth（あれば）→ モデル値 が使われます。

### RESORTS.snowDepth について

- HTML 内の `RESORTS` に **snowDepth** が入っているゲレンデは、`resort-snow.json` にその ID が無い場合に「ゲレンデ公表」としてこの値が使われます。
- 元データの更新頻度に依存するため、日々の積雪更新には `resort-snow.json` を更新する運用の方が向いています。
